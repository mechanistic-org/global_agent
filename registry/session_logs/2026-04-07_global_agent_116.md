## Decisions
- Elected to utilize `docling` for Office and PDF asset unification due to its local PyTorch backend layout engine, preserving structural fidelity (tables, headers) without relying on cloud APIs.
- Adopted `pandas` to process raw spreadsheet data with an aggressive 1000-row ingest clamp, protecting the agent from token overflow.
- Designed the `normalize_local_asset` MCP tool to return absolute file paths instead of loading the extracted Markdown into the raw JSON-RPC stream, forcing ephemeral storage in `.tmp/normalized/` (or a custom deployment path) to prevent MCP payload bloating.
