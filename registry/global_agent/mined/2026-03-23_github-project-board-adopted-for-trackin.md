The existing GitHub Project board will be leveraged for tracking LinkedIn content pieces, using a `[Content]` prefix for issues. These content tasks will be sized as `Task` and categorized by `Node` (`portfolio` or `global_agent`) and `Impact` (`Revenue` or `Aesthetic`).
