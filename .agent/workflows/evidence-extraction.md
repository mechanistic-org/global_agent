# Workflow: Evidence Extraction & GitHub Closure (The R2 Bridge)

This workflow defines the rigorous, automated process for converting local agent debug recordings (like browser stress tests) into permanent, live forensic evidence attached to a closed GitHub Issue without any "drag-and-drop" human intervention.

It leverages the **Asset Sovereignty** (R2 bridge) architecture and the **GitHub CLI** (`gh`) to create a high-fidelity, permanent paper trail for your C-Suite/Engineering Manager personas on `eriknorris.com`.

### The Core Loop
When an Agent completes a visual or systemic verification task (e.g., creating a `.webp` browser recording), it MUST execute the following steps to formally close the loop:

1. **Extract to Sovereignty** (Move the file out of the `.gemini/brain` and into the R2 Staging Drive):
```powershell
// turbo
Move-Item "C:\Users\erik\.gemini\antigravity\brain\$CONVERSATION_ID\*.webp" "D:\GitHub\eriknorris-assets\R2_STAGING\forensics\"
```

2. **Format the Live URL** (Construct the immutable production path):
The agent must mentally calculate the future live URL based on the `mootmoat` R2 proxy routing constraints.
*   **Format:** `https://assets.eriknorris.com/forensics/[filename].webp`

3. **Construct the Final Walkthrough**:
The `walkthrough.md` generated by the agent MUST embed the image using the newly calculated **Live URL**, not the local `C:\` path.
*   `![Forensic Verification](https://assets.eriknorris.com/forensics/[filename].webp)`

4. **Append & Close via Native CLI** (Update the GitHub record):
The agent must use the native `gh` CLI to append the walkthrough as the final comment and immediately close the issue, ensuring the timeline is perfectly preserved.
```powershell
// turbo
& "C:\Program Files\GitHub CLI\gh.exe" issue comment $ISSUE_NUMBER -R eriknorris/eriknorris --body-file "C:\Users\erik\.gemini\antigravity\brain\$CONVERSATION_ID\walkthrough.md"
& "C:\Program Files\GitHub CLI\gh.exe" issue close $ISSUE_NUMBER -R eriknorris/eriknorris
```

**Why this matters:**
This is the heart of high-fidelity mining. We are not just fixing bugs; we are autonomously building a public, verified, and permanent database of robust engineering practices (The Mined Artifacts) that ATS parsers and technical hiring authorities can audit.
