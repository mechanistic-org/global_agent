# 📦 Asset Sovereignty & The Dual Proxy Protocol (MootMoat)

## When to use this skill
- When you encounter a **404 error** for an image or asset.
- When configuring Astro API routes or Astro SSR adapters.
- When the user asks to "sync assets", "push images", or handle media.

## The Architecture (The Dual Proxy)
To prevent massive memory leaks in the Vite/Astro framework, the MootMoat ecosystem does **NOT** store heavy video/audio media (like the 34MB `.m4a` files) inside the `public/` directory or the git repository. 

All heavy media is fetched dynamically via the proxy route.
This proxy has two strict physical states:

1. **Local Development (`import.meta.env.DEV`):**
   - The proxy uses `node:fs` and `node:path` to read files directly from the local external drive: `D:\GitHub\mootmoat-assets\R2_STAGING`.
   - *Constraint:* You must never `await import("node:fs")` at the top level of the API route, as Cloudflare Workers will crash on compilation. It must be dynamically imported inside the `if (DEV)` block.

2. **Production (`import.meta.env.PROD`):**
   - The proxy uses the `env.MOOTMOAT_ASSETS.fetch` Cloudflare Worker binding to pull the image from the `mootmoat-assets` R2 Bucket.
   - *Constraint:* Cloudflare Workers do not have access to a local filesystem.

## The Enforcement Laws

### 1. Pre-Flight Enforcement (No Local Symlinks)
- Do **NOT** create symlinks or junctions to `public/assets`.
- All `<img>`, `<audio>`, or Markdown media tags for heavy files MUST point to the proxy route. Never hardcode `public/` for heavy binaries.

### 2. The Truth Pipeline (Troubleshooting 404s)
- If an asset is 404ing locally, verify it exists in `D:\GitHub\mootmoat-assets\R2_STAGING`.
- If an asset is 404ing in production (Cloudflare), it implies the `mootmoat-assets` bucket is out of sync with the local staging drive.
- **Action:** If the bucket is out of sync, execute the user workflow `/sync_assets` to push the local files to R2 via the sync utility.

### 3. Synthesis Override
Assets may be generated by the AI **ONLY** for:
- Generic UI placeholders (textures, patterns) that live lightly in `public/`.
You shall **NOT** generate bespoke, heavy product media. Rely on the user's R2 drive for truth.
